# Implementation Safety

Use this overlay when `Coding Pack` is designing, implementing, or reviewing a change.

Minimum checks:

- The change boundary matches the active spec and design.
- Important risks, regressions, or exceptions are explicit.
- Rework paths are preserved when review finds issues.
- The current implementation state is understandable by another contributor.

If the pack cannot explain safety and scope, it should not claim the work is ready to advance.
