# Closeout Readiness

Use this overlay when the pack is deciding whether current work can leave the active coding mainline.

Minimum checks:

- Review and verification state are attached to the closeout summary.
- Residual risks, follow-ups, and unresolved gaps are visible.
- The current outcome is classified instead of implied.
- Another contributor can understand why the work stopped here.

If the answer is still "keep building", route back into active work instead of pretending the slice is finished.
