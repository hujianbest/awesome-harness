# Input Completeness

Use this overlay before `Coding Pack` accepts a bridge or scoped request into the coding mainline.

Minimum checks:

- The input states the problem, target, and scope boundary.
- Acceptance criteria or intended outcome are visible.
- Important unknowns, risks, and missing pieces are explicit.
- The pack can explain why this input is ready for coding now.

If these checks fail, hold intake or return upstream for clarification instead of forcing implementation.
