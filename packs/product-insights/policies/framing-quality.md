# Framing Quality

Use this overlay when a `Product Insights Pack` node is deciding whether the current framing is stable enough to continue.

Minimum checks:

- The problem statement is explicit and bounded.
- The target user or operating context is named.
- The current goal and non-goals are separated.
- Open questions are visible instead of hidden in prose.

Hold the flow if the pack still relies on ambiguous problem wording or unstated scope.
