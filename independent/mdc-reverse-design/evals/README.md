# 独立版代码逆向设计补档评测

这个目录包含独立版 `mdc-reverse-design` 的评测 prompts。

## 目的

这些评测用于验证该 skill 是否真正做到：

- 在逆向补档前先判断代码范围、入口和上下文是否足够
- 在提供真实代码夹具时，优先基于文件证据提炼设计，而不是只复述 prompt
- 在仓库范围过大时先切补档边界，而不是直接写整仓失真总设计
- 从代码中提取设计层信息，而不是退化为文件遍历或实现讲解
- 明确区分 `代码事实`、`设计推断` 和 `待确认`
- 在用户指定路径或章节骨架时优先遵循，而不是回落默认模板
- 在用户施加“直接补全完整设计”压力时，仍然保持事实优先
- 在用户实际要的是目标态设计时，能识别这不是 reverse-design 的正确阶段

其中：

- 正向能力场景应尽量依赖 `evals/fixtures/` 下的真实代码夹具
- 门禁或阶段边界场景可以不带文件，用来验证它是否会正确停止或回退

## 建议评分关注点

1. 是否先判断当前输入是否足以支撑可靠补档。
2. 是否在大仓库场景下先做范围切分或严格收口到指定模块。
3. 是否把输出保持在设计层抽象，而不是写成代码导读、类图讲解或任务拆解。
4. 是否把关键结论明确标成 `代码事实`、`设计推断` 或 `待确认`，而不是混为一谈。
5. 是否在存在冲突、缺失或低置信度区域时显式暴露，而不是强行补齐。
6. 是否遵循用户给定的落盘路径与章节结构。
7. 是否在旧草稿修订场景下保留有效结构，而不是整份重写。
