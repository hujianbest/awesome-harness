## 1. owner question

人的方向、规则和审批怎样真正进入团队工作主线，而不是停留在口头约定里。

## 2. 关键判断

- governance 是产品能力，不是附属品
- 规则、gate、approval、archive 必须是显式 artifacts
- governance 约束 session、execution、growth，但不替代它们

## 3. 非目标

- 不把治理退化成 prompt 习惯
- 不把治理 runtime 变成新的业务执行层
