# D31: Growth Memory Skill Experience

- Design ID: `D31`
- 状态: 草稿
- 日期: 2026-04-11
- 定位: 定义用户如何感知 `memory`、`skill`、growth proposal 与团队长期成长体验。
- 关联文档:
  - `docs/architecture/21-evidence-and-continuity-layer.md`
  - `docs/architecture/31-growth-and-evolution-layer.md`
  - `docs/features/F140-continuity-and-growth.md`

## 1. owner question

用户如何感知团队因为做过的事情而真正变强，而不是只看到黑箱学习。

## 2. 设计判断

- `memory`、`skill`、`proposal` 必须在产品层可区分
- growth 必须可解释
- 长期成长必须能回指来源 evidence 与治理过程

## 3. 产品层要求

- memory visibility
- skill visibility
- proposal visibility
- accepted growth visibility

## 4. 不负责什么

- 不定义 promotion runtime
- 不定义具体 store 结构
