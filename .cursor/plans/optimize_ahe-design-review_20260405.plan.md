# 优化 `ahe-design-review` 方案

## 目标

让 `ahe-design-review` 更稳定地判断设计是否已达到任务规划前的质量门。

## 当前短板

- 触发时机还可以更明确。
- 测试准备度与任务级测试细案之间的边界可以更清楚。
- 设计评审容易被误用成任务规划建议入口，需要更强的非目标提示。

## 计划中的实际改动

- 收紧 description
- 增加适用时机
- 增加非目标 / 不做事项
- 收紧测试准备度边界

## 预期效果

- 更稳定地把设计评审限制在设计层
- 更少把任务级细节提前带进设计评审
